### Architecture

#### Storage
https://docs.0g.ai/concepts/storage
- Can be used without using the blockchain - useful for decentralized storage
- Appears that storage is sharded - not every node will have full copy of storage data
- There are two "storage lanes"
	- **Publishing lane:** metadata and availability proofs, verified with 0G consensus
	- **Storage lane:** Actual storage
		- Erasure coding allows up to 30% of nodes to fail
			- According to docs, nodes will automatically rebalance to maintain redundancy levels for erasure coding
- Storage providers get rewards based on "Proof of Random Access" or PoRA
	- system randomly asks miners to provide a hash of some data
- Miners (which store data apparently) must store data in 8TB chunks
	- This is for fairness - larger miners can simply have more 8TB chunks
